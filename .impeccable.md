# Recodex Web Design Context

## Target Audience

Recodex Web is used by developers running `recodex-go` locally. They open the bundled web console to inspect the local Bridge, pair the mobile app, and understand workspace/session/Git state without logging in.

## Core Use Cases

- Show a QR code that the mobile Flutter app can scan to pair with the local Bridge.
- Display connected devices, available workspaces, session history, and Git status/actions.
- Make empty or offline states understandable when the backend has not been restarted or has no registered data yet.
- Keep Git write operations clear and deliberate while the web console itself remains unauthenticated.

## Brand Personality

Calm, precise, local-first developer tooling. The interface should feel trustworthy and operational, with refined spacing, clear hierarchy, and a small amount of product character from the Flutter app icon. Avoid marketing-page drama and avoid authentication/pairing language for the web console.

## Quality Bar

Flagship bundled console: polished enough to ship with `recodex-go` as the default local visualization UI.
